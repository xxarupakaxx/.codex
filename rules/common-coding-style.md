# Common Coding Style Rules

全言語・全プロジェクトに適用される共通コーディング規約。

## イミュータビリティ

- 変数は可能な限り`const`/`final`/`let`（再代入不要な場合）で宣言
- オブジェクトの直接変更よりスプレッド演算子/コピーを優先
- 配列の`push`より`concat`/スプレッド（関数型アプローチ）

## ファイル構成

- 1ファイル200-400行を目安（最大800行）
- 1ファイル1責務（Single Responsibility）
- 関数は80行以内を目安
- ネストは最大3レベル（早期リターンで制御）
- 既存ファイル内のスタイル（クォート種別/type hint有無等）に合わせる（Style Drift防止）

## エラーハンドリング

- 汎用`catch`でエラーを握りつぶさない
- エラーメッセージにはコンテキスト情報を含める
- リトライが必要な場合はexponential backoff
- 外部API呼び出しには必ずタイムアウトを設定

## 入力バリデーション

- システム境界（ユーザー入力、外部API）では必ずバリデーション
- 内部コード間の呼び出しでは型システムを信頼
- バリデーションエラーは具体的なメッセージで返す

## コード品質チェックリスト

変更前に確認:
- [ ] 早期リターンパターンが使われているか
- [ ] マジックナンバー/マジックストリングがないか
- [ ] 重複コードがないか
- [ ] 関数名・変数名は意図を表現しているか
- [ ] 不要なコメント（コードの繰り返し）がないか
