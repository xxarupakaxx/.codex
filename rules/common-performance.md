# Common Performance Rules

全プロジェクトに適用されるパフォーマンス規約。

## データベース

- N+1クエリを避ける（eager loading / JOIN を使用）
- 頻繁に検索されるカラムにはインデックスを付与
- 大量データはページネーションで取得（LIMIT/OFFSET or カーソル）
- `SELECT *` を避け、必要なカラムのみ取得

## API・ネットワーク

- 不要なAPIコールを減らす（キャッシュ、バッチ処理）
- レスポンスには適切なCache-Controlヘッダー
- 大きなペイロードは圧縮（gzip/brotli）
- API呼び出しは可能な限り並列化（Promise.all等）

## フロントエンド

- 不要な再レンダリングを避ける（memo, useMemo, useCallback）
- 画像は適切なサイズ・フォーマット（WebP/AVIF）
- 遅延読み込み（lazy loading）を活用
- バンドルサイズを意識（tree shaking、dynamic import）

## メモリ

- 大きなオブジェクトの参照を不要になったら解放
- イベントリスナーは適切にクリーンアップ
- ストリーム処理で大量データを扱う（全件メモリに載せない）

## アルゴリズム

- O(n²)以上のアルゴリズムには注意（Map/Setで高速化を検討）
- ループ内での重い処理を避ける
- 再帰は深さに注意（スタックオーバーフロー）
