# Notes

## 運用原則

計画は実装を拘束する仮説であり、観察不能な未来を確定する文書ではない。

実行後の観察で計画を変える場合は、元の outcome を消さず、Revision Log に理由と再検証先を追加する。

変更によって acceptance を弱める場合は人間の再承認が必要である。

## 用語

- **Outcome Trace**：成果と要件、実装、合格基準、証拠、人間確認の対応。
- **Revision Event**：観察によって計画を変えた記録。
- **Human Review**：機械検証では判断できない見た目、理解可能性、運用妥当性の確認。
- **Objection**：成果を承認する前に読み手が持つ可能性のある反論。
