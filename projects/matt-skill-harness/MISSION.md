# Mission

## Learning goal

利用者がロードマップを開いて一分以内に、次の五点を説明できる状態にする。

1. 何を実現する計画か。
2. どこまで実装されたか。
3. どの checkpoint が未達か。
4. 何を見れば人間が確認できるか。
5. 実行後の観察で計画が変わった箇所はどこか。

## Current level

Outcome Trace、Sprint Contract、Team Journal の概念は理解している。

しかし、別ファイルへ分散した情報を読む負荷が高く、最初に見る場所が明確ではない。

## Success criteria

- 最初の画面だけで Now、Next human decision、Coverage、unresolved objections が読める。
- 各 outcome から requirement、implementation、acceptance、evidence、human review を横に追える。
- missing state と failed holistic check を色だけに頼らず区別できる。
- 計画変更が元の計画を上書きせず、観察、変更理由、再検証先として残る。
- teach lesson の小テストで、漏れと計画変更を正しく判定できる。
