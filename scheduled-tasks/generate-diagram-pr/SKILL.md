---
name: generate-diagram-pr
description: 毎時、対象リポジトリのオープンPRに状態図を冪等に投稿/更新（マーカーコメントで重複防止）
---

【目的】毎時実行。対象リポジトリのオープンPRに、状態図（generate-state-diagram スキル）で生成した図をコメント投稿する。図があると変更がわかりやすいため。既にある場合は新規投稿せず既存コメントを更新する（冪等）。

【設定読み込み】
- `~/.claude/config/user.json` を Read。対象リポジトリは `github.watch_repos`（配列）があればそれ、無ければ既定で `ydb-superapp-server`。

【手順】
1. 対象リポジトリのオープンPRのうち、直近24時間以内に作成/更新されたものを `gh` で取得する（古い・stale なPRは対象外）。1回の実行で最大10PRまで（過負荷防止）。
2. 各PRについて、generate-state-diagram スキルでブランチ変更内容からMermaid状態図/処理フロー図を生成する。
3. 【冪等性ガード】各PRの既存コメントを `gh` で取得し、マーカー文字列 `<!-- claude-generate-diagram -->` を含むコメントを検索する:
   - 存在する → そのコメントを編集（gh pr comment --edit-last 等）で図を更新する。新規コメントは作らない。
   - 存在しない → マーカー文字列を先頭に含めた新規コメントとして図を投稿する。
4. コメント本文は必ず1行目に `<!-- claude-generate-diagram -->` を含め、次回以降の検索・更新の目印にする。

【注意】
- 図に変化が無い（前回と同一）場合は更新もスキップしてノイズを減らす
- 巨大PR（数十ファイル超）は主要な状態遷移に絞って図示する
