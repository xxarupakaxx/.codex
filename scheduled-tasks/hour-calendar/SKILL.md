---
name: hour-calendar
description: 毎時、直近1時間に終了したミーティングの議事録を要約して日次ノートに追記し、ネクストアクションを分類・自律実行する
---

【目的】毎時実行。直近1時間に「終了した」ミーティングの議事録/文字起こしを拾い、日本語で日次ノートに詳細に追記し、ネクストアクション（やること）を分類・自律実行する。

【出力先（最初に確定する）】
- `~/.claude/config/user.json` を Read し、`notes.daily_dir` を取得する（未設定/ファイル不在なら `~/.claude/.local/daily/` を使う）。
- 当日(JST)の日次ノートを `{daily_dir}/{YYYY-MM-DD}.md` とする。無ければ作成し、先頭に `# {YYYY-MM-DD} 日次ノート` 見出しを置く（既存内容は決して消さない、追記のみ）。

【手順】
1. 現在時刻を JST(Asia/Tokyo) で取得し、「実行時刻の60分前〜実行時刻」を対象ウィンドウとする。終了済みの予定のみ対象（開催中・未開催は除外）。
2. Google カレンダー(primary) で対象ウィンドウのイベントを取得する。
3. 各イベントについて議事録/文字起こしの所在を確認する：添付ファイル（「Gemini によるメモ」「メモ - 〈イベント名〉」等の Google ドキュメント）/ 説明欄のリンク（Google ドキュメント / Confluence の議事録ページ）。ローカルにファイルがある前提にしない。中身が空・テンプレだけの予定は対象外。
4. メモ/文字起こしが存在するミーティングだけ本文を読み込む。
5. 【冪等性ガード】当日ノートを Read し、対象ミーティングの見出し（`## [HH:MM] <イベント名>`）が既に存在すればスキップする（二重追記しない）。
6. ミーティングごとに日本語で詳細にまとめ、当日ノート末尾へ追記する：
   - 見出し: `## [HH:MM] <イベント名>`
   - 概要（1〜2行）: 何の打ち合わせか／誰と
   - 主要な論点（詳細）: 議論の中身・背景・出た結論や判断
   - 出た話題・雑談: 余談やアイスブレイクのトピック
   - AIの使い方: 会話に出た AI ツール／活用法・気づき
   - やること（ネクストアクション）: 自分のTODO・宿題・次回までの確認事項（担当/期限が分かれば併記）
7. 【秘匿・センシティブ除外（厳守）】パスワード・トークン・会議パスコード等は一切書かない。1on1 等での個人評価・人事・処遇など他者に関するセンシティブな内容は記録しない。共有ノートなら社外秘の機能名・数値・未公開情報は一般化して書く。

【ネクストアクションの分類・実行】
8. 手順6の「やること」を3分類する：
   - AIタスク(auto_execute): 「調べてまとめる」=リサーチ+ドキュメント生成 / 「コードを確認」=コードベース調査 / 「PRをレビュー」=PRレビュー / 「仕様書を書く」=仕様書ドラフト / 「Jiraに起票」=チケット作成
   - 人間タスク(human_action): 人への連絡 / 日程調整 / 意思決定・承認 / 本番デプロイ / 金額・契約・人事に関わるもの
   - 情報(info_only): 共有された決定事項 / 背景情報 / 次回ミーティングの議題
9. 【分類の安全ルール（厳守）】曖昧なアクション項目は必ず human_action に倒す。金額・契約・人事・本番操作は必ず human_action。
10. 【自律実行ポリシー — 信頼タスク全自律】auto_execute は確認なしで Agent Tool 等を使い自律実行してよい（このタスクは信頼タスク扱い）。ただし安全のため次を厳守：
    - 同じアクションを二重実行しない（当日ノートに `[AI実行済み]` の記録があるものはスキップ）
    - 外部への起票/投稿/コメントは冪等に行う（既存のチケット・コメント・PR・ドキュメントを先に検索し、あれば更新、無ければ作成）
    - 実行結果を当日ノートに追記する（例: `[AI実行済み] 〇〇を調査 → Confluenceに投稿 <URL>`）
11. human_action は当日ノートの `## TODO` セクションに追記する。期限が当日のものがあれば Slack（user.json の slack.notification_channel、無ければ DM）でリマインドする。
12. 直近1時間にメモ付きミーティングが無ければ、何も出力せず静かに終了する。
