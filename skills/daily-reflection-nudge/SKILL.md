# /daily-reflection-nudge — ふりかえりの白紙をなくす

Dailyのふりかえり欄は空欄が続きやすい（診断時点で連日未記入）。
ゼロから書くより下書きを直す方が負荷が低いので、このスキルはその日の記録から材料を集めて下書きを置く。
**まず `CLAUDE.md` を読み、絶対ルール（追記のみ、Claude-note不可侵）に従うこと。** 地の文は `japanese-tech-writing` の規範で書く。

入力: $ARGUMENTS

## 1. 対象日の決定と発動条件

- $ARGUMENTS に日付があればその日。なければ**今日**（夜間実行で日付をまたいだ直後は前日を選んでよい）。
- 対象Dailyの「🔁 ふりかえり」3行（👍 / 💡 / ➡️）がすべて未記入のときだけ発動する。1行でも本人の記入があれば、そのDailyには触れない（本人の言葉を上書きしない）。

## 2. 材料の集め方

その日の記録だけを材料にする。創作しない。

- その日に作成されたノートの `summary`（`rg "^date_created: <日付>"` で列挙）
- `payment-trend` の `learning_theme` と `trend` の「🧭 所感」
- Dailyの「✅ タスク」の完了状況と「💭 メモ」
- digestの「概要」

## 3. 下書きの書き方

各行の下に、4スペース字下げのサブ項目として**（AI下書き）ラベルつき**で1〜3個を追記する。

```md
- 💡 気づき・学び:
    - （AI下書き）payment-trendの学習テーマ: 〜。正本は [[11_B2B支払い・請求・承認・消込・会計連携]]。
```

- 👍 よかったこと: 完了した作業や回った学習ループなど、その日の記録から確認できる事実。
- 💡 気づき・学び: 学習テーマ、知見ノートのsummary、発見した問題。根拠ノートへ [[リンク]] する。
- ➡️ 明日やること: 未回答のクイズ、未処理の確認事項、Dailyの未完了タスクから、翌日に持ち越すもの。

本人の一人称の感想（楽しかった、疲れた等）は書かない。
下書きは事実と導線に徹し、感情と評価は本人の編集に委ねる。

## 4. 報告

追記した下書きの行数と対象Dailyへのリンクを短く返す。
本人が下書きを編集または削除したら、それが正になる（（AI下書き）ラベルの行は本人がいつ消してもよい使い捨て）。

## ガード

- ふりかえり欄への追記は（AI下書き）ラベルつきサブ項目のみ。既存の記入がある欄には追記しない。
- ジャーナル欄（📝）には書かない（OCR書き起こしと本人の日記の領域のため）。
- リネーム・削除はしない。`Claude-note/` は読み取りのみ。

## ⏰ スケジュール設定

- **主モードは on-demand**。定期化するなら夕方〜夜に1日1回（例: `0 21 * * *`）が目安。登録は人間が [[SCHEDULES]] の手順で行う。
- まず [[今日ダッシュボード]] を開く習慣で自然に埋まるかを試し、埋まらない日が続くようなら定期化する。
