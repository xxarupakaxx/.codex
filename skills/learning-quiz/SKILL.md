# /learning-quiz — 読んだつもりを潰す想起クイズ

読み直しより想起（思い出す練習）のほうが記憶に定着する。
このスキルは、合格基準つきの学習ノートを答え合わせの正本として、本文を見ずに答えるクイズを対話で回す。
**まず `CLAUDE.md` を読み、絶対ルール（追記のみ、新規はInbox配下、Claude-note不可侵）に従うこと。**

入力: $ARGUMENTS

## 1. スコープと出題対象の決定

- $ARGUMENTS でスコープ指定（`payment` / `簿記` / 特定ノート名）。無指定なら **payment**。
- 出題対象ノートの優先順:
  1. 過去クイズで「復習メモ」が残っているノート（`rg "## 復習メモ" Inbox/knowledge/` で検出）。再出題が最優先。
  2. 未出題のノート。過去の出題履歴は `Inbox/automation/quiz/quiz-*.md` の `related` から機械的に把握する。
  3. どちらもなければ [[Paymentドメイン基礎マップ]] の読む順（簿記なら [[簿記3級 はじめに（目次と全体像）]]の目次順）。
- 1セッション2〜3ノート、5問前後。多くしない（採点と振り返りの質を優先）。

## 2. 出題の作り方

対象ノートを読み、[[00_Paymentドメイン基礎ノート 合格基準]] の観点から問いを作る。
1セッション内で観点を散らす。

- **一文定義**（そのテーマを一文で言えるか）
- **登場人物と責任分界**（誰が何を担うか）
- **お金とデータの流れ**（誰から誰へ、いつ、何が動くか）
- **なぜ生まれたか / 何を解いたか**
- **例外ケース**（失敗、差し戻し、証憑不足で誰が何をするか）
- **LayerX/Bakurakuへの接続**（どの顧客ペイン、どの業務に効くか）

問いは選択式にせず、説明させる形式にする（再認でなく想起を試すため）。
答えの要点は出題ノートの本文だけを根拠にし、本文にない知識を正解に含めない。

## 3. クイズノートの生成

`Inbox/automation/quiz/quiz-YYYY-MM-DD.md` を作成する（同日2回目は追記）。

- frontmatter: `type: note`、`tags: [automation, quiz, layerx-prep]`、`summary`、`depth: overview`、`related` に**出題元ノート**（これが出題履歴の機械可読データを兼ねる）。
- 本文構成: 問いの一覧 → 各問いの下に折りたたみの答え（`> [!note]- 答えの要点` 形式。根拠ノートへの [[リンク]] つき） → `## 回答記録`（採点後に追記）。
- 実行日の `Daily/YYYY-MM-DD.md` の「💭 メモ」へ `- 🧠 想起クイズ: [[quiz-YYYY-MM-DD]]` を追記。

## 4. 対話での採点と記録（ユーザーが回答したら）

1. 回答を出題ノートの本文と突き合わせ、**○（言えた）/ △（部分的）/ ×（言えなかった）** で採点する。根拠の節名を添えて短くフィードバックする。
2. クイズノートの `## 回答記録` に日付、問い、判定、弱点の1行を追記する。
3. △と×の問いは、**出題元ノートの末尾に「## 復習メモ」節を追記**（既にあれば行を追記）して残す。書式:
   `- [ ] YYYY-MM-DD 問い: <言えなかった問い>（quiz-YYYY-MM-DD）`
4. 復習メモの問いに次回のクイズで答えられたら、その行のチェックを `- [x]` に更新してよい（行は消さない）。

未回答のままセッションが終わってもよい。クイズノートは残るので、後日開いて自分で答え合わせできる。

## 5. 報告

出題した問いの数、対象ノート、クイズノートへのリンクを短く返す。採点まで行った場合は○△×の内訳と復習メモの追記先も返す。

## ガード

- 出題と採点の根拠は出題ノート本文に限る。本文にない「一般知識としての正解」で採点しない（ノート側が誤っている疑いが出たら、採点でなく「ノートの要修正候補」として報告する）。
- 既存ノートへの変更は「復習メモ」節への追記と、復習メモ行のチェック更新のみ。
- `Claude-note/` は対象外。リネーム・削除はしない。

## ⏰ スケジュール設定

- **主モードは on-demand**（学習の気分になったときに呼ぶ）。定期化するなら平日夜または週末に週2〜3回が目安（登録は人間が [[SCHEDULES]] の手順で行う）。ただし無人実行では採点相手がいないため、出題ノートの生成までに留まる。
