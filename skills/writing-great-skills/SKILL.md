スキルは、確率的なシステムから決定性を引き出すために存在する。
**予測可能性**とは、同じ出力ではなく、エージェントが毎回同じ**プロセス**を取ることであり、根本的な価値である。
以下の手段はすべて、それを支える。

**太字の用語**は [`GLOSSARY.md`](GLOSSARY.md) で定義している。
完全な意味はそちらで確認する。

## 起動

異なるコストを交換する二つの選択肢がある。

- **モデル起動**スキルは **description** を保持する。
  そのため、エージェントが自律的に起動でき、ほかのスキルからも到達できる。
  人間が名前を入力して起動することもできる。
  description が毎ターンのウィンドウに入るため、**コンテキスト負荷**が生じる。
  `disable-model-invocation` を省き、「ユーザーが何を望む場合、何に言及した場合に使うか」のように、エージェント向けの起動表現を豊富に含む description を書く。
- **ユーザー起動**スキルは、エージェントから description を隠す。
  人間が名前を入力した場合だけ起動でき、ほかのスキルからも到達できない。
  コンテキスト負荷はゼロだが、存在を覚える索引を**人間**が担うため、**認知負荷**を支払う。
  `disable-model-invocation: true` を設定する。
  `description` は人間向けの一行の要約とし、起動フレーズの一覧を除く。

エージェントが自力でスキルへ到達する必要がある場合、またはほかのスキルから到達する必要がある場合だけ、モデル起動を選ぶ。
手動でしか起動しないならユーザー起動にして、コンテキスト負荷をゼロにする。

ユーザー起動スキルが人間の記憶範囲を超えて増えた場合は、その認知負荷を**ルータースキル**で解消する。
ほかのスキルの名前と使う時機を示す、ユーザー起動スキルを一つ作る。

## Description を書く

モデル起動スキルの **description** には、スキルが何であるかを示すことと、起動すべき**分岐**を列挙することという二つの役割がある。
すべての語が**コンテキスト負荷**を増やすため、本文以上に厳しく削る。

- **スキルの先導語を冒頭に置く。**
  description では、先導語が起動に作用する。
- **分岐ごとに起動条件を一つだけ置く。**
  同じ分岐を言い換える同義語は**重複**である。
  「TDD で機能を作る」と「テストファースト開発を依頼する」は同じ分岐を二度書いている。
  一つにまとめ、真に異なる分岐だけを残す。
- **本文にすでにある自己紹介を削る。**
  description には起動条件と、必要なら「ほかのスキルが必要とする場合」の到達条件だけを残す。

## 情報階層

スキルは、自由に組み合わせられる**ステップ**と**参照情報**という二種類の内容から構成する。
すべてがステップ、すべてが参照情報、その両方のいずれでもよい。
どちらを使うか、各内容を**情報階層**のどこへ置くかを決める。
情報階層は、エージェントが情報を必要とする即時性で順位付けしたものである。

1. **スキル内のステップ**：`SKILL.md` に置く順序付きの操作であり、主要な層である。
   各ステップは、作業の完了を示す**完了基準**で終える。
   完了かどうかを判断できるように**検査可能**にする。
   必要な場合は、「変更したモデルをすべて説明する」のように**網羅的**にする。
   「変更一覧を作る」のような曖昧な基準は、**早すぎる完了**を招く。
2. **スキル内の参照情報**：`SKILL.md` 内で必要に応じて参照する定義、ルール、事実である。
   レビューの全ルールを同じ段へ置くような、妥当な同格集合になることも多く、問題ではない。
   このスキルは全体が参照情報である。
3. **外部の参照情報**：`SKILL.md` の外へ移し、**コンテキストポインター**を通じて必要な場合だけ読み込む参照情報である。
   スキルの一部である `GLOSSARY.md` のような開示ファイルから、スキルシステム外の完全な**外部参照情報**までを含む。

要求水準の高い完了基準は、徹底した**下準備**を促す。
これはステップの有無に依存しない。
「すべてのルールを適用する」は、平坦な参照情報へも適用されるためである。

下へ移す情報が少なすぎると上部が肥大化し、多すぎると実際に必要な情報が隠れる。
この緊張関係を判断する。

**段階的開示**とは、`SKILL.md` の内容をリンク先ファイルへ移し、階層を下げることである。
これにより上部の読みやすさを保つ。
スキルフォルダに、内容を表す名前の `.md` ファイルを置いてリンクする。
このスキルは完全な定義を `GLOSSARY.md` へ開示している。
スキルを使う方法ごとに異なる**分岐**があり、実行ごとに経路が変わる場合がある。
分岐は開示を判断する最も明確な基準である。
すべての分岐が必要とする情報はインラインへ置き、一部の分岐だけが到達する情報はポインターの後ろへ移す。
**コンテキストポインター**がいつ、どれほど確実に情報へ到達させるかは、リンク先ではなく文言で決まる。

情報階層が情報を**どこまで下げるか**を決めるのに対し、**併置**は下げた場所で**何を隣に置くか**を決める。
概念の定義、ルール、注意点を一つの見出しの下へまとめ、ファイル全体に散らさない。
一部を読めば、関連する情報も近くで読めるようにする。

## 分割する条件

**粒度**はスキルを分ける細かさである。
分割するたびに二種類の負荷のどちらかを支払うため、効果がある場合だけ分ける。

- **起動による分割**：単独で起動すべき固有の**先導語**がある場合、またはほかのスキルから到達する必要がある場合に、**モデル起動**スキルを分ける。
  常に読み込まれる **description** が増えるため、**コンテキスト負荷**を支払う。
  独立した到達性に、そのコスト以上の価値が必要になる。
- **順序による分割**：現在のステップに続く**完了後のステップ**が、目の前のステップを急がせて**早すぎる完了**を招く場合に、一連の**ステップ**を分ける。
  後続を見えなくすることで、現在の作業により多くの**下準備**を行うよう促す。

## 剪定

各意味を**信頼できる唯一の情報源**へ置く。
振る舞いを変えるときに、一か所だけ編集すればよい状態にする。

各行の**関連性**を検査する。
その行は今もスキルの動作に関係しているかを問う。

次に、行単位だけでなく文単位で**無作用**を探す。
各文を単独で検査し、無作用なら語を削って整えるのではなく、文全体を削除する。
積極的に削る。
検査に失敗する文章の多くは、書き直すのではなく取り除くべきである。

## 先導語

**先導語**は、モデルの事前学習にすでに存在し、スキルの実行中にエージェントが考えるために使う簡潔な概念である（例：_lesson_、_fog of war_、_tracer bullets_）。
本文全体で繰り返すと、分散した定義が積み上がり、事前知識を呼び起こして最小限のトークンで広い振る舞いを固定する。
ただし、強い先導語なら一度だけで足りる場合もある。

先導語は予測可能性へ二つの効果を与える。
本文では**実行**を固定し、語が現れるたびに同じ振る舞いを選ばせる。
description では**起動**を固定する。
同じ語がプロンプト、文書、コードにも存在すると、エージェントは共有語彙をスキルへ結び付け、より安定して起動する。

先導語を使うようにスキルをリファクタリングできる箇所を探す。
三か所で説明される三要素の組や、description の一文を使って一つの概念を示している箇所は、単一のトークンへ**圧縮**できる候補である。

- 「fast, deterministic, low-overhead」から _tight_ へ変更する。
  一つのフェーズで繰り返される性質を、事前学習済みの一語（_tight_ loop）へまとめる。
- 「a loop you believe in」から _red_ へ変更する。
  曖昧なゲートを二値で観測できる状態へ変える。
  ループがバグによって _red_ になるか、ならないかで判断できる。

トークンが減り、エージェントが思考を掛けるフックも明確になるという二つの効果がある。
すべてのスキルに、先導語で置き換えられる言い換えがあると仮定して探す。

## 失敗モード

ユーザーがスキルで困っている問題を診断するときに使う。

- **早すぎる完了**：ステップが本当に終わる前に、作業ではなく完了することへ注意が移って終了する。
  最初に完了基準を明確にする。
  これは局所的でコストが低い。
  曖昧さを除けず、実際に急ぐ挙動を観測した場合だけ、順序を分割して完了後のステップを隠す。
- **重複**：同じ意味が複数の場所にある。
  保守コストとトークンを増やし、情報階層で本来の順位以上に意味を目立たせる。
- **堆積**：追加は安全で削除は危険だと感じるため、古い層が積み重なる。
  剪定の規律がないスキルが行き着く既定の状態である。
- **肥大化**：すべての行が現役で固有でも、スキルが単純に長すぎる。
  可読性と保守性を損ない、トークンを浪費する。
  参照情報をポインターの後ろへ開示し、分岐または順序で分割して、各経路が必要なものだけを持つようにする。
- **無作用**：モデルが既定ですでに従うため、負荷を支払っても何も変えない行である。
  既定と比べて振る舞いを変えるかを検査する。
  エージェントがすでにある程度徹底的なら、弱い先導語 _be thorough_ は無作用になる。
  手法を変えるのではなく、_relentless_ のような強い語へ変える。
- **否定**：禁止による誘導は逆効果になる。
  _don't think of an elephant_ は象をコンテキストへ持ち込み、利用可能性を高める。
  **肯定形**で目標の振る舞いを書き、禁止対象へ触れない。
  肯定形にできない強いガードレールとしてだけ禁止を残し、その場合も行うべきことを併記する。
