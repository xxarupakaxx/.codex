# 日報テンプレート

## 出力フォーマット

```markdown
# 日報 YYYY-MM-DD（曜日）

## 今日やったこと

### [PJ名] タスク名
- 実施内容の概要
- 主な変更点・作業内容

### [PJ名] タスク名2
- ...

## 成果・完了したもの

- [x] 完了したタスクや成果物（コミット、PR、マージ等）
- [x] ...

## 課題・問題点

- 遭遇した問題や未解決の課題
- ブロッカーとなっている事項
- （なければ「特になし」）

## 明日のTODO

- [ ] 未完了タスク（40_progress.mdから抽出）
- [ ] 新たに発生したタスク
- [ ] 後回しにした項目（99_history.mdから抽出）

## ふりかえり（KPT）

### Keep（良かったこと・続けたいこと）
- 効率的だったプロセスや良い判断

### Problem（問題・改善が必要なこと）
- 時間がかかった作業、非効率だった点

### Try（次に試すこと）
- 改善案、新しいアプローチ
```

## 抽出ルール

### 05_log.md からの抽出

- `**ユーザー指示:**` ブロック → タスクの目的を把握
- `**レスポンス:**` ブロック → 実施内容を抽出
- Phase情報があれば進行状況を判断

### 30_plan.md からの抽出

- `## タスク一覧` → 計画されたタスクの全体像
- 各タスクの完了/未完了ステータス

### 40_progress.md からの抽出

- `## 完了タスク` → 成果セクションに反映
- `## 進行中タスク` → 明日のTODOに反映
- `## 未着手タスク` → 明日のTODOに反映
- `## 発生した問題` → 課題セクションに反映

### 80_review.md からの抽出

- レビュー指摘の要約 → 課題セクションに反映（重要なもののみ）

### 99_history.md からの抽出

- 後回し判断 → 明日のTODOに反映（再開条件を含む）

## KPT生成のヒント

### Keep候補
- 短時間で完了したタスク
- レビューで指摘が少なかったもの
- 過去知見（learnings-researcher）が役立った場面

### Problem候補
- 長時間かかったタスク
- レビューで多くの指摘を受けたもの
- 計画からの乖離
- ブロッカーで待ちが発生した場面

### Try候補
- Problemに対する具体的な改善案
- 使えそうな新しいツール・アプローチ
- プロセス改善のアイデア

## 複数PJ時の構成

PJが複数ある場合:

```markdown
## 今日やったこと

### [project-a] 認証機能の実装
- ...

### [project-a] バグ修正 #123
- ...

### [project-b] DB マイグレーション
- ...
```

PJ名はディレクトリ名の末尾（basename）を使用。
