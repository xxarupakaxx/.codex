新しいエージェントが作業を続けられるように、現在の会話の引き継ぎ要約を書く。
要約を保存する代わりに、要約をプロンプトとして渡したバックグラウンドエージェントを起動する：`claude --bg --name "<descriptive name>" "<handoff summary>"`。
エージェントは現在の作業ディレクトリで起動し、コマンドはすぐに制御を返す。
ユーザーは `claude agents` でエージェントを管理する。

必ず説明的な名前を `-n` または `--name` で渡す（例：`--name "Fix login bug"`）。
この名前はジョブ一覧、セッション選択画面、ターミナルタイトルに表示される。

要約に「推奨スキル」セクションを含め、エージェントが起動すべきスキルを提案する。

PRD、計画、ADR、issue、コミット、diff など、ほかの成果物に記録済みの内容を重複させない。
代わりにパスまたは URL を参照する。

API キー、パスワード、個人を特定できる情報などの機密情報は伏せる。
要約はエージェントのプロンプトになるためである。

ユーザーが引数を渡した場合は、次のセッションで取り組む内容の説明として扱い、それに合わせて要約を調整する。
