# ビルドエラー修正ワークフロー

## 概要

ビルドエラー・型エラー・コンパイルエラーを**最小限の変更**で修正する。
根本原因を特定し、安全かつ効率的に修正を適用する。

## 手順

### 1. エラーの収集と分類

まずビルドコマンドを実行してエラーを全て収集:

```bash
# TypeScript
npx tsc --noEmit 2>&1 | head -100

# または PJ固有のビルドコマンド
npm run build 2>&1 | head -100
```

エラーを以下に分類:
- **型エラー** (TS2xxx): 型の不一致、未定義のプロパティ等
- **インポートエラー** (TS2307等): モジュール解決の失敗
- **構文エラー**: 文法上の問題
- **ランタイム依存**: 外部パッケージの問題

### 2. 依存関係の解析

エラー間の依存関係を特定:
- 1つの修正で複数のエラーが解消されるケースを優先
- カスケードエラー（1つの根本原因から派生するエラー群）を識別

### 3. 修正の適用

**原則: 最小変更**

- 型エラー → 型の修正 or 型ガードの追加（`as`キャストは最終手段）
- インポートエラー → パスの修正、依存パッケージの追加
- 構文エラー → 該当箇所のみ修正
- 修正ごとにビルドを再実行して確認

### 4. 検証

```bash
# ビルド確認
npm run build

# 型チェック
npx tsc --noEmit

# テスト（既存テストが壊れていないか）
npm test
```

## 禁止事項

- `// @ts-ignore` や `// @ts-expect-error` で抑制すること
- `any` 型で逃げること
- エラーと無関係なコードを変更すること
- ビルド設定（tsconfig.json等）の`strict`を緩めること

## 優先順位

1. 根本原因の修正（1つ直せば複数解消されるもの）
2. インポート・依存関係の修正
3. 個別の型エラー修正
4. 残りの軽微なエラー
