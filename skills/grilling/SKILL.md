# Grilling

目的は質問数ではなく、実行前の共通理解である。
決定木の依存を一枝ずつ解消する。

1. **事実と意思決定を分ける。**
   - 事実は codebase に限らない。環境、設定、既存成果物、利用可能なツールから local-first で確認する。
   - ローカル確認で不足し、独立した証拠を作る価値がある場合だけ Delegation Gate を通して調査を委任する。
   - ユーザーの選好、優先順位、許容リスク、採否は事実として推定しない。
2. **意思決定を一問ずつ聞く。**
   - 一度に複数の質問を投げない。
   - 各質問に推奨、根拠、主要な trade-off を添える。
   - 依存する質問は、その前提となる決定から聞く。
   - 回答間の矛盾は、次の質問として明示する。
3. **確認で閉じる。**
   - 決まったこと、未決事項、捨てた選択肢を簡潔に要約する。
   - ユーザーが共通理解を確認するまで、計画の実行、文書更新、ADR、tracker 操作、外部操作を始めない。

`grill-me` はこの skill の user-facing wrapper である。
合意を durable artifact にする必要があると確認された場合だけ `grilling-with-docs` を使う。
