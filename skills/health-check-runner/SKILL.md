# /health-check-runner — 月次点検を1コマンドにする

[[Second Brain ヘルスチェック]] は点検項目を定義しているが、手で開いて確認する運用は続かないことが記録から分かっている（初回2026-06のみ）。
このスキルは機械で測れる項目をスクリプトで測り、人間は結果を読んで判断だけする分担にする。
**まず `CLAUDE.md` を読み、絶対ルール（追記のみ、新規はInbox配下、Claude-note不可侵）に従うこと。**

## 1. 機械実行する点検項目

対象は `Inbox/knowledge/` 配下（ダッシュボードと同じスコープ）。判定はスクリプトで行い、目視で数えない。

1. **出典なし候補**: frontmatterに `url` / `source` / `evidence_urls` がなく、本文にも出典系の見出し（出典、根拠URL、参考、公式リンク）がないノート。自分の考えだけのノートは問題ではないので、件数と一覧を「確認候補」として出すに留める。
2. **孤立ノート**: アウトリンク0本のノート。
3. **MOC到達性**: 4つのMOCからの距離3以上のノート（判定方法は [[moc-gardener]] と同じ無向BFS）。
4. **as_of鮮度**: `as_of` が6か月以上前のノート。
5. **拡張スキーマ充足**: `summary` / `related` / `depth` の欠落数。あわせてスキャン系出力（digest、trends、payment-trends）の直近ノートが新スキーマを持つかを確認する。
6. **矛盾・重複の機械検出は限定的**: summaryが酷似するノートの組だけ挙げ、内容の矛盾判断は人間項目として残す。

## 2. 記録の残し方

1. 詳細ノート `Inbox/automation/health/health-check-YYYY-MM-DD.md` を作成する（拡張スキーマつき。一覧はここに全件書く）。
2. [[Second Brain ヘルスチェック]] の「## 記録」節に、月見出し（`### YYYY-MM（機械実行 YYYY-MM-DD）`）で件数サマリと詳細ノートへのリンクを**追記**する。既存の記録は書き換えない。
3. 点検項目のチェックボックス自体は人間の月次儀式なので機械では触らない。

## 3. 報告

各項目の件数、悪化した項目（前回記録との比較）、詳細ノートへのリンクを短く返す。
修正が必要な項目は、対応するスキル（[[moc-gardener]]、[[vault-metadata-migrator|vault-metadata-migrator]]）への導線を添える。

## ガード

- 既存ノートへの変更は [[Second Brain ヘルスチェック]] の記録節への追記のみ。検出した問題の修正はこのスキルでは行わず、該当スキルに委ねる。
- `Claude-note/` は対象外。リネーム・削除はしない。

## ⏰ スケジュール設定

- **主モードは on-demand**。定期化するなら月1回、月初が目安（登録は人間が [[SCHEDULES]] の手順で行う）。
