新しいエージェントが作業を続けられるように、現在の会話を要約した引き継ぎ文書を書く。
文書は現在のワークスペースではなく、ユーザーの OS の一時ディレクトリへ保存する。

文書には、エージェントが起動すべきスキルを提案する「推奨スキル」セクションを含める。

仕様書、計画、ADR、issue、コミット、diff など、別の成果物に記録済みの内容は重複させない。
代わりにパスまたは URL を参照する。

API キー、パスワード、個人を特定できる情報などの機密情報は伏せる。

ユーザーが引数を渡した場合は、次のセッションで取り組む内容の説明として扱い、それに合わせて文書を調整する。
