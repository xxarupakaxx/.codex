# Wizard を生成する

**wizard** は、手作業では面倒で、毎回 AI に説明し直すのも面倒な手順を、人間へ一段ずつ案内する bash スクリプトである。
各 URL を開き、クリックしてコピーする対象を正確に伝え、値を取得し、適切な場所（`.env`、GitHub secrets）へ書き込み、各段階で確認し、残りの量を表示する。
サードパーティーサービスの設定、一度限りの移行、プロジェクトをある状態から別の状態へ移す作業などに使える。

使いやすい UX は [template.sh](template.sh) ですでに用意されている。
残り時間付きの進捗表示、確認ゲート、WSL を含むクロスプラットフォームの URL オープン、secret の非表示入力、冪等な `.env` の upsert、`gh secret` と `gh variable` の書き込み、終了時の要約を備える。
**担当するのは、手順の範囲を決め、各 stage を書くことだけである。**
`STAGES` マーカーより上のライブラリはすべての wizard で同一であり、その一貫性に価値がある。
手で編集しない。

wizard は既定では一時的な成果物である。
一度の実行用に scratch または `scripts/` のパスへ作り、作業が終わったら削除する。
リポジトリに残す反復可能なセットアップ手順をユーザーが望む場合だけコミットする。

## プロセス

### 1. 手順の範囲を決める

人間が行うすべての手順と、途中で取得するすべての値を特定する。
質問から始めず、最初にリポジトリを読む。

- セットアップの場合：`.env`、`.env.example`、`.env.*`、`README`、`docker-compose*`、フレームワークの設定、`.github/workflows/*` を読む。
  `secrets.*` と `vars.*` への参照は、すべて wizard が生成すべき値である。
- 移行または遷移の場合：現在の状態、目標の状態、その間にある不可逆な操作を調べる。

次に、stage の順序付き一覧と各 stage が生成する値をユーザーへ示し、確認する。
ユーザーは追加、削除、並べ替えを行える。

**完了条件：**すべての stage に順番と名前があり、取得する各値について、（a）人間がどこで取得するか、（b）どこへ書くか（`.env`、GitHub secret、両方、または書き込まない。操作だけの stage もある）、（c）secret として非表示入力するか、公開値かがわかっている。

### 2. 各 stage の経路を対応付ける

各 stage について、開く URL、そこで行う操作、値が表示される場所、値を入れる変数を含め、人間がたどる正確な経路を書く。
例：「Dashboard → Developers → API keys → Reveal test key → copy」。
現在の UI や正確なコマンドを実際には把握していない場合は、そのことを伝え、ユーザーへ尋ねるか文書を確認する。
存在しない可能性がある手順を捏造しない。

**完了条件：**すべての stage が、初見の人でも実行できる具体的な手順へ対応している。

### 3. wizard を作成する

`template.sh` を対象のパスへコピーする。
例の stage を、依存順に並べた手順ごとの `stage` へ置き換える。
ライブラリの helper である `stage`、`say` / `step`、`open_url`、`ask` / `ask_secret`、`write_env`、`set_secret` / `set_var`、`pause` / `confirm` を使う。
`TOTAL_STAGES` と `TOTAL_MINUTES` には正直な見積もりを設定する。
これらが残り時間の表示に使われる。

テンプレートが示す基準を守る。
値を尋ねる前に URL を開き、secret には `ask_secret` を使い、永続化するすべての値へ `write_env` を使う。
`set_secret` は CI が実際に必要とする値だけに使い、不可逆な操作の前には `confirm` を置く。
各 `stage` は画面を消去し、現在の手順だけを表示する。
人間が必要とする情報が画面外へ流れないように、一つの stage を一つの作業へ絞る。
マーカーより上のライブラリには触れない。

### 4. 検証して引き渡す

- `bash -n <script>` を実行する。
  利用可能なら `shellcheck` も実行する。
- `chmod +x <script>` を実行する。
- エージェント自身で最後まで実行しない。
  ブラウザーを開き、人間の入力を待つためである。
  代わりに静的に追跡する。
  手順1のすべての値が取得され、指定した場所へ書き込まれ、すべての `set_secret` 名が CI の `secrets.*` 参照と完全に一致することを確認する。
- 実行方法をユーザーへ伝える。
  反復可能なセットアップ手順ならコミットし、README からリンクする。
  次の作業者が AI へ尋ねずスクリプトを実行できるようにする。
