---
name: jira-spec-poll
description: 毎時、自分アサインの新規Jiraチケットを検出し仕様書ドラフトを自動生成
---

【目的】毎時実行。自分にアサインされた新規チケットを検出し、仕様書ドラフトを生成する。

【手順】
1. Jira MCPで自分アサインの最近更新されたチケットを検索する:
   - JQL: `assignee = currentUser() AND statusCategory != Done AND updated >= -1h ORDER BY updated DESC`
   - 直近1時間以内に更新（アサイン含む）されたチケットが対象

2. 各チケットについて、既に仕様書が作成済みかチェックする（冪等性）:
   - チケットのコメントに「仕様書ドラフト」というタグがあれば既存とみなしスキップ
   - `~/.claude/.local/specs/` 配下に `{チケットキー}.md` があればスキップ

3. 新規チケットごとに仕様書ドラフトを生成する:
   - チケットの説明・受入基準・コメント・関連チケットを読み込む
   - コードベースの関連箇所をGrep/Globで調査
   - 以下のテンプレートで仕様書を生成:
     - 概要 / 背景 / 機能要件 / 非機能要件 / 技術設計 / テスト計画 / リスク / サブタスク候補

4. 生成した仕様書を保存する:
   - `~/.claude/.local/specs/{チケットキー}.md` に保存
   - チケットにコメントとして追加（「仕様書ドラフト — レビューをお願いします」）

5. Slackで自分に通知する:
   - 「{チケットキー} の仕様書ドラフトを生成しました。Jiraコメントを確認してください。」

【注意】
- 仕様書はドラフト（レビュー前）としてマークする
- セキュリティ関連の変更は必ず「リスク」セクションに記載
- 1回の実行で最大3チケットまで処理（過負荷防止）
- 既存の仕様書がある場合は上書きしない
